    ユーザー作成¶

    1. MySQL5.7までの状況 
      GRANT 権限 ON データベース名.テーブル名 TO ‘ユーザー名’ IDENTIFIED BY ‘パスワード’;
    2. MySQL8.0からこう変わった 
      CREATE USER ‘ユーザー名’ IDENTIFIED BY ‘パスワード’;でユーザーを作成
GRANT 権限 ON データベース名.テーブル名 TO ‘ユーザー名’;で権限を付与

    authentication_policy¶

    authentication_policyの指定がなければ、デフォルトの設定になります。¶

    authentication_policyを指定なし 
    # authentication_policyの指定がなければ、デフォルトの設定になります。
CREATE DATABASE `mydb`;
CREATE USER 'myuser'@'%' identified BY 'password';
GRANT ALL PRIVILEGES ON mydb.* TO 'myuser'@'%';
FLUSH PRIVILEGES;
select user,host,plugin from mysql.user where user = 'myuser';

    authentication_policyを指定してユーザの作成も可能です。¶

    authentication_policyを指定 
    # authentication_policyを指定してユーザの作成も可能です。
CREATE DATABASE `mydb`;
CREATE USER 'myuser'@'%' identified with caching_sha2_password BY 'password';
GRANT ALL PRIVILEGES ON mydb.* TO 'myuser'@'%';
FLUSH PRIVILEGES;
select user,host,plugin from mysql.user where user = 'myuser';

    root ユーザのパスワード忘れて、ログイン出来ないとき¶

    // 一度終了
$ mysql.server stop

// セーフモードでの起動、
$ mysqld_safe --skip-grant-tables &

// パスワードなしでログインできる
$ mysql -u root

// root のパスワードを空にする
mysql > UPDATE mysql.user SET authentication_string=null WHERE User='root';
mysql > exit;

// MySQL再起動（非セーフモード）
$ mysql.server restart

// 空のパスワードでログイン
$ mysql -u root -p 

// 
mysql > USE mysql;
mysql > ALTER USER 'root'@'localhost' identified BY 'hoge';
