    Thanosは、Prometheusのデータを長期間保管し、大規模な監視システムを構築するためのオープンソースプロジェクトです。Thanosは複数のコンポーネントから成り立っており、それぞれが特定の役割を担っています。以下に、主要なコンポーネントの用途を説明します。

    1. Thanos Sidecar¶

    • 用途: Thanos Sidecarは、実行中のPrometheusサーバーに隣接して動作します。その主な役割は、Prometheusが収集したデータをリアルタイムで読み取り、Thanosシステムの他の部分がアクセスできるようにすることです。さらに、SidecarはPrometheusのTSDBブロックをリモートストレージ（例：S3、GCSなど）にアップロードすることもできます。これにより、データの長期保管とスケーラビリティが向上します。

    2. Thanos Query¶

    • 用途: Thanos Queryコンポーネントは、複数のThanos SidecarやThanos Storeからデータを集約してクエリするフロントエンドとして機能します。ユーザーはThanos Queryを通じて、分散された監視データに対して統一的なクエリを実行できます。これにより、システム全体の可視性が向上し、データの分析が容易になります。

    3. Thanos Store¶

    • 用途: Thanos Storeは、リモートストレージ（S3、GCSなど）に保存された歴史データに対する長期ストレージのゲートウェイとして機能します。Storeコンポーネントは、これらの長期データをThanos Queryに提供し、クエリの実行を可能にします。Thanos Storeは、大量の監視データを扱う際に、効率的なデータアクセスを実現します。

    4. Thanos Compact¶

    • 用途: Thanos Compactは、リモートストレージに保存されたデータブロックをより効率的に管理するためのコンポーネントです。これは、複数の小さいデータブロックを大きなブロックに統合（コンパクション）し、クエリのパフォーマンスを向上させると同時に、ストレージ使用量を最適化します。また、Thanos Compactはデータのダウンサンプリングも行い、異なる解像度でのデータビューを提供します。

    これらのコンポーネントは合わさって、Thanosが提供する強力なデータ保管とクエリの機能を実現します。モジュール式の設計により、Thanosは柔軟に拡張可能であり、小規模な環境から大規模なクラスタまで、さまざまな監視システムのニーズに対応できます。

    Thanos SidecarがS3（またはS3互換のオブジェクトストレージ）にデータを保存するタイミングは、主にPrometheusのブロックの生成とアップロードの周期に依存します。Prometheusは通常、2時間ごとにデータのブロックを生成し、これらのブロックはThanos Sidecarによって自動的にS3にアップロードされます。以下は、そのプロセスの概要です：

    1. ブロックの生成: Prometheusは監視データを時間単位で区切り、2時間ごとにデータブロックとしてディスクに書き込みます。これらのブロックには、収集されたメトリクスデータが格納されています。
    2. ブロックのアップロード: Thanos Sidecarは、Prometheusが新しいブロックを生成すると、それを検出し、S3にアップロードします。このプロセスは、ブロックが作成されるたびに自動的に行われます。
    3. コンパクション: Prometheusには、時間が経つにつれて小さなブロックをより大きなブロックに統合するコンパクションプロセスがあります。Thanosでは、Thanos CompactorコンポーネントがS3に保存されたブロックのコンパクションを担います。このプロセスにより、長期間のクエリのパフォーマンスが向上し、ストレージの使用効率が改善します。
    4. 即時性：Thanos Sidecarは、Prometheusサーバーからリアルタイムでメトリクスを収集し、Thanosクエリレイヤーを通じてクエリ可能にします。しかし、S3（または他のオブジェクトストレージ）に永続的にデータが保存されるのは、ブロックが生成されアップロードされた後です。

    Thanos Sidecarを使う利点の一つは、この自動的なアップロードプロセスによって、監視データの長期保管とスケーラビリティが向上することです。ただし、リアルタイムデータとS3にアップロードされるデータ間のわずかな遅延は、このプロセスの性質上発生します。

    また、Thanos Sidecarの設定オプションを通じて、この挙動に影響を与えることが可能です。例えば、アップロード間隔を調整することで、どのくらいの頻度でデータがS3に送信されるかを制御できます。しかし、基本的なアップロードのタイミングは、Prometheusのブロック生成周期に従います。

#!/bin/sh

set -e

iptables -F
iptables -X
iptables -P INPUT ACCEPT
iptables -P OUTPUT ACCEPT
iptables -A INPUT -i lo -j ACCEPT
iptables -I INPUT -p icmp --icmp-type 0 -j ACCEPT
iptables -I INPUT -p icmp --icmp-type 8 -j ACCEPT
iptables -A INPUT -m state --state ESTABLISH,RELATED -j ACCEPT

iptables -A INPUT -s 172.16.0.0/12 -j ACCEPT
iptables -A INPUT -s 192.168.0.0/16 -j ACCEPT

iptables -A INPUT -j DROP

/sbin/iptables-save > /etc/sysconfig/iptables

services:
  prometheus:
    image: prom/prometheus
    container_name: prometheus
    user: root
    volumes:
      - /home/prometheus:/etc/prometheus  # prometheus の設定ファイル場所
      - /home/prometheus/data:/prometheus # prometheus のデータ保存場所
    command:
      - "--config.file=/etc/prometheus/prometheus.yaml"
      - "--web.enable-admin-api"          # HTTP API 許可

      - default
